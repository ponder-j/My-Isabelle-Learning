## 结构化证明的几种开头

| `proof` 后的方法       | 含义和用途                                                                        |
| :----------------- | :--------------------------------------------------------------------------- |
| **`-`**            | **什么都不做**。提供一个“原始”的证明环境，用于完全手动的证明。                                           |
| **(无)**            | **默认行为**。应用标准、安全的初始规则，通常是最好的起点。                                              |
| **`(induct ...)`** | **归纳法**。为归纳证明设置好框架。                                                          |
| **`(cases ...)`**  | **分情况讨论**。将证明分解为多个独立的子情况。                                                    |
| **`(rule ...)`**   | **应用规则**。从一个特定的定理或引理开始证明。                                                    |
| **`(simp ...)`**   | **简化**。在证明开始前先进行一轮代数化简。                                                      |
| **`(auto ...)`**   | **自动化**。尝试用更强的自动化工具解决或简化目标。                                                  |
| **`(clarify)`**    | **清理**。安全地简化目标，使其更清晰，但不做有风险的步骤。比如把 $\longrightarrow$ 自动换成 $\Longrightarrow$  |
